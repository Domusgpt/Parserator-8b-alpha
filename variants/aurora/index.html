Inside the stack

How the Architect → Extractor pipeline keeps trust intact

Parserator sequences deterministic steps with LLM arbitration so teams receive structured JSON they can inspect and override at any point.

Plan

Architect builds the SearchPlan

Gemini 1.5 Flash ingests the schema and raw text, mapping every field to a sourcing strategy the Extractor can follow.

  • Deterministic instructions stored alongside the request
  • Telemetry reveals why each field was sourced
Execute

Extractor executes deterministic passes

Pattern libraries, validators, and cached lookups harvest confident fields instantly, shrinking the work that hits an LLM.

  • Token savings by skipping redundant prompts
  • Consistent latency for dashboards and automations
Arbitrate

LLM arbitration fills the edges

Only the unresolved spans move to arbitration, where Parserator double-checks extractor output and surfaces confidence scores.

  • Confidence and token telemetry returned in every response
  • Fallback paths to request human or agent review

Lean agent toolkit

ParseratorCore keeps every deployment portable

The same Architect → Extractor engine that powers the API, dashboard, and SDK can run locally. Sessions, plan caches, and profile presets make it simple to adapt Parserator to new agents without losing telemetry.

Plan once, reuse anywhere

Cache SearchPlans with the bundled createInMemoryPlanCache or swap in your own store so parseMany and sessions skip redundant architect calls across batches.

  • Portable via session.exportInit()
  • API, SDK, and local cores share the cache

Sessions with guardrails

ParseratorSession captures confidence history, streams telemetry, and auto-refreshes plans when getAutoRefreshState() spots dips—ideal for high-volume inboxes.

  • Seed once, parse many documents
  • Telemetry mirrors production responses

Profiles & interceptors

Apply presets like lean-agent or sensor-grid with applyProfile, then wire analytics or governance hooks via core.use() for every parse.

  • Swap Architect/Extractor implementations
  • Expose diagnostics to custom listeners

Operational guardrails

Diagnostics make every parse traceable

The production stack exposes the same metadata everywhere: REST responses, the Node SDK, and the dashboard share confidence scores, token counts, and lean fallback telemetry so stakeholders never lose observability.

Confidence metadata

Responses return metadata.confidence, token spend, and field:fallback streams so you can tell when the deterministic Extractor finished the work and when the lean LLM bridge helped.

  • Inspect telemetry in API, SDK, and dashboard views
  • Reason about every field’s origin in audits

Session snapshots

ParseratorSession captures history with session.snapshot(), triggers autoRefresh when confidence dips, and exposes getAutoRefreshState() so operators know exactly when plans regenerated.

  • Keep plan caches aligned across teams
  • Export snapshots to satisfy compliance checks

Lean queue health

Async resolvers broadcast queue metrics, cooldown windows, and last error information via getLeanOrchestrationSnapshot(), making it clear when fallbacks ran and why.

  • Share queue health with Clear Seas dashboards
  • Turn telemetry into alerts with custom listeners

Integrate fast

Drop Parserator into your pipeline in minutes

Choose between the hosted REST endpoint or the published Node SDK. Both expose the same metadata, safeguards, and guarantees.

Send JSON via REST

POST the data you want parsed alongside your desired schema. Parserator replies with structured JSON plus metadata describing every extraction decision.

  • Include schema + raw text in one call
  • Confidence and token telemetry in the response
  • Backed by 95% accuracy and ~2.2s latency
curl -X POST "https://app-5108296280.us-central1.run.app/v1/parse" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer pk_live_your_api_key" \
  -d '{
    "inputData": "Acme Co. Contract Signed 3/12/24",
    "outputSchema": {
      "company": "string",
      "event": "string",
      "date": "date"
    }
  }'
