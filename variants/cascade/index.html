Follow the production workflow

The two-stage pipeline is documented across the README, architecture briefs, and EMA white paper so teams can evaluate Parserator with the same telemetry we use internally.

  1. Stage 01

    Architect plans the parse

    Gemini 1.5 Flash maps entities, schema bindings, and risky spans up front to minimise downstream retries while surfacing decision traces.【F:claude/architecture.md†L1-L112】

  2. Stage 02

    Extractor applies deterministic rules

    The heuristic stack resolves most fields locally through regex, normalisers, and validators before any LLM escalation.【F:docs/HEURISTIC_STACK_REVIEW.md†L1-L120】

  3. Stage 03

    Telemetry closes the loop

    Responses include confidence scores, token usage, and trace IDs so operators can replay or override edge cases.【F:claude/architecture.md†L1-L112】

Integrate Parserator quickly

Choose between the REST endpoint and the Node SDK—the pipeline and telemetry stay identical.

Send JSON via REST

POST raw transcripts and the schema you expect. Parserator replies with structured JSON plus metadata covering every extraction decision.

  • Include schema + raw text in one call
  • Confidence + token telemetry returned
  • 95% accuracy · ~2.2s latency
curl -X POST "https://app-5108296280.us-central1.run.app/v1/parse" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer pk_live_your_api_key" \
  -d '{
    "inputData": "Acme Co. Contract Signed 3/12/24",
    "outputSchema": {
      "company": "string",
