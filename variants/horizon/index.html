Operational playbook for ParseratorCore

Navigate the controls ops teams rely on to keep plan caches fresh, sessions accountable, and lean fallbacks in view.

Keep long-running sessions honest

Launch createSession, monitor session.snapshot(), and inspect session.getPlanState() to watch plan health in real time.

  • session.refreshPlan() rebuilds instructions when confidence drops.
  • session.getAutoRefreshState() documents when auto-refresh fired.

Scale batches without re-planning

Cache Architect output with createInMemoryPlanCache and fan work out through parseMany so repeated schemas skip planning costs.

  • session.exportInit() keeps plan data portable across queues.
  • API, SDK, and local cores share the same cache contract.

Audit lean fallbacks like a dashboard

Subscribe to field:fallback telemetry, poll getLeanLLMFieldFallbackState(), and use getLeanOrchestrationSnapshot() to track queue cooldowns.

  • Telemetry hub mirrors ParseService events for local or remote deployments.
  • Confidence + token metrics stay attached to every response for audits.

Integrate Parserator your way

Choose REST or the Node SDK—both expose telemetry and EMA safeguards.

REST endpoint

POST raw text + schema and get structured JSON with confidence scores and token telemetry.

curl -X POST "https://app-5108296280.us-central1.run.app/v1/parse" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer pk_live_your_api_key" \
  -d '{
    "inputData": "Acme Co. Contract Signed 3/12/24",
    "outputSchema": {
      "company": "string",
      "event": "string",
      "date": "date"
    }
  }'

Node SDK

Install the published SDK to manage retries, telemetry metadata, and confidence scoring with minimal setup. 

npm install parserator-sdk

import { ParseratorClient } from 'parserator-sdk';

const client = new ParseratorClient({
  apiKey: 'pk_live_your_api_key_here'
});

const result = await client.parse({
  inputData: `John Smith\nSenior Developer\njohn@techcorp.com\n(555) 123-4567`,
  outputSchema: {
    name: 'string',
    title: 'string',
    email: 'email',
    phone: 'phone'
  }
});

console.log(result.parsedData);
