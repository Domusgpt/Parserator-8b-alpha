Plan

Architect builds the SearchPlan

Gemini 1.5 Flash maps every requested field to sourcing tactics the Extractor can execute deterministically.

  • Instructions saved with the request for auditing
  • Reason codes surface for every extraction
Execute

Extractor handles deterministic passes

Validators, pattern libraries, and cached lookups capture most fields without touching an LLM.

  • Latency stays predictable for dashboards
  • Token spend drops across large batches
Arbitrate

LLM fills the edges

Only unresolved spans move to arbitration where Parserator double-checks outputs and attaches confidence scores.

  • Confidence + token telemetry returned in every response
  • Fallback hooks for human or agent review

Lean core for agents and operators

ParseratorCore ships the Architect → Extractor pipeline as a portable module you can embed anywhere.

Reusable SearchPlans

Cache Architect output with the bundled createInMemoryPlanCache and reuse it across parseMany batches, API calls, and sessions.

  • session.exportInit() keeps plans portable
  • Shared cache between API and SDK

Sessions & auto-refresh

ParseratorSession exposes snapshots, telemetry, and getAutoRefreshState() so ops teams know when plans regenerate after confidence dips.

  • Seed once, stream many parses
  • Confidence + token history on tap

Profiles & interceptors

Flip behaviours with applyProfile presets such as lean-agent or plug analytics through core.use() interceptors.

  • Swap Architect/Extractor implementations
  • Expose diagnostics to custom observers

Operational guardrails stay in view

Parserator’s telemetry surfaces make every Architect plan and Extractor decision auditable—no matter if you’re inside the API, the Node SDK, or Clear Seas dashboards.

Confidence telemetry

Responses ship metadata.confidence, token counts, and field:fallback events so teams can see when lean LLM support engaged and why.

  • Aligns with dashboard and ParseService logs
  • Field-level provenance for compliance reviews

Session snapshots

ParseratorSession records session.snapshot() history, fires autoRefresh when confidence dips, and exposes getAutoRefreshState() for live status.

  • Keep plan caches synchronized across agents
  • Export snapshots to share during audits

Lean queue visibility

Async resolvers broadcast queue metrics, cooldown timers, and last-error metadata via getLeanOrchestrationSnapshot() so you know exactly when fallbacks ran.

  • Monitor fallback health without leaving Clear Seas
  • Route telemetry into alerts with custom listeners

Integrate Parserator your way

Choose REST or the Node SDK—both expose telemetry and EMA safeguards.

REST endpoint

POST raw text + schema and get structured JSON with confidence scores and token telemetry.

curl -X POST "https://app-5108296280.us-central1.run.app/v1/parse" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer pk_live_your_api_key" \
  -d '{
    "inputData": "Acme Co. Contract Signed 3/12/24",
    "outputSchema": {
      "company": "string",
      "event": "string",
      "date": "date"
    }
  }'

Node SDK

Install the published SDK to manage retries, telemetry metadata, and confidence scoring with minimal setup. 

npm install parserator-sdk

import { ParseratorClient } from 'parserator-sdk';

const client = new ParseratorClient({
  apiKey: 'pk_live_your_api_key_here'
});

const result = await client.parse({
  inputData: `John Smith\nSenior Developer\njohn@techcorp.com\n(555) 123-4567`,
  outputSchema: {
    name: 'string',
    title: 'string',
    email: 'email',
    phone: 'phone'
  }
});

console.log(result.parsedData);
