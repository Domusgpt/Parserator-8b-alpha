How it stays explainable

Two-stage workflow with inspectable checkpoints

Parserator’s modular stages make it easy to trace every extraction decision, adjust heuristics, and re-run parses without rebuilding orchestration.

01

The Architect

Gemini 1.5 Flash inspects a small sample plus your schema, producing a reusable SearchPlan that flags ambiguities before extraction begins.

  • Reusable profiles
  • Schema aware
  • Telemetry-ready
02

The Extractor

Deterministic heuristics resolve most fields locally. Lean LLM calls only fire when the plan asks for assistance, reducing spend while maintaining accuracy.

  • Section-aware parsing
  • Deterministic fallbacks
  • Confidence scores
03

Telemetry & audit trail

Every request emits structured metadata so operators, agents, and customers can verify outcomes or rerun extractions with updated heuristics at any time.

  • Confidence reporting
  • Replay-ready
  • Freedom to leave

Send JSON via REST

POST the data you want parsed alongside your desired schema. Parserator replies with structured JSON plus metadata describing every extraction decision.

  • Include schema + raw text in a single call
  • Confidence and token telemetry in the response
  • Backed by 95% accuracy and ~2.2s latency

Lean agent toolkit

Bring the Architect → Extractor kernel anywhere

ParseratorCore is the portable heart of the platform. Drop it into local agents, swap heuristics, or mirror production telemetry without inheriting the Firebase stack.

Reusable SearchPlans

Cache Architect output once and replay it across workloads. The built-in plan cache and parseMany helper reuse SearchPlans so repeated schemas skip planning costs.

  • createInMemoryPlanCache ships in core
  • session.exportInit() keeps plans portable

Session guardrails

Long-lived ParseratorSessions hold confidence snapshots, auto-refresh plans when scores dip, and surface getAutoRefreshState() so ops teams know when recalibration fired.

  • Seed inputs once, reuse across batches
  • Telemetry mirrors production responses

Profiles & interceptors

Tune heuristics with applyProfile presets like lean-agent or register interceptors via core.use() to hook analytics, guardrails, or custom audit logs into every parse.

  • Swap Architect/Extractor implementations
  • Expose diagnostics through custom listeners

Operational guardrails

Telemetry keeps every parse accountable

Confidence metadata, queue diagnostics, and session snapshots ship with Parserator across the API, Node SDK, and MCP surfaces so ops teams always know when the Architect planned, when the Extractor resolved, and when the lean fallback assisted.

Confidence & token traces

Responses expose metadata.confidence, token usage, and field:fallback telemetry, mirroring what the dashboard and ParseService log so you can audit every extraction decision.

  • Surface telemetry in REST, SDK, and dashboard flows
  • Inspect lean fallback attempts and skip reasons

Session snapshots

Long-lived ParseratorSessions record session.snapshot() states, auto-refresh SearchPlans when confidence dips, and expose getAutoRefreshState() so you can track when recalibration triggered.

  • Keep plan caches in sync across agents and API
  • Replay snapshots during audits or rollbacks

Queue instrumentation

The lean resolver uses async task queues that emit health metrics, cooldown timers, and last-error metadata via getLeanOrchestrationSnapshot() so you know exactly when the LLM bridge engaged.

  • Monitor fallback queues without leaving Clear Seas
  • Share diagnostics downstream through custom listeners
