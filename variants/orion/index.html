Workflow

Architect → Extractor timeline

Parserator keeps the EMA “freedom to leave” promise by exposing every transformation stage. Each step below ships today inside the production API and dashboard.

  • 01

    Architect plans the parse

    Gemini 1.5 Flash identifies entities, schema bindings, and risky spans before a single token touches the Extractor cache.【F:claude/architecture.md†L1-L112】

  • 02

    Extractor executes deterministically

    The Extractor applies heuristics, regex stacks, and schema validators so the majority of fields resolve without additional LLM calls.【F:docs/HEURISTIC_STACK_REVIEW.md†L1-L120】

  • 03

    Telemetry + review hooks

    Responses ship with confidence scores, token usage, and trace IDs so teams can replay ambiguous parses or override with domain rules.【F:claude/architecture.md†L1-L112】

Send JSON via REST

POST the data you want parsed alongside your desired schema. Parserator replies with structured JSON plus metadata describing every extraction decision.

  • Include schema + raw text in a single call
  • Confidence and token telemetry in the response
  • Backed by 95% accuracy and ~2.2s latency
curl -X POST "https://app-5108296280.us-central1.run.app/v1/parse" \
  -H "Content-Type: application/json" \

  -d '{
    "inputData": "Acme Co. Contract Signed 3/12/24",
