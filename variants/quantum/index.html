How the pipeline protects trust

Architect and Extractor align deterministic work before arbitration ever touches an LLM.

Plan · Architect

Gemini 1.5 Flash builds the SearchPlan, mapping schema fields to deterministic sourcing tactics.

  • Instructions stored with the request
  • Reason codes ready for audits

Execute · Extractor

Pattern libraries, validators, and cached lookups capture most fields instantly.

  • Predictable latency across workloads
  • Token savings from skipping redundant prompts

Arbitrate · LLM

Only unresolved spans go to arbitration, which double-checks outputs and adds confidence scores.

  • Telemetry returns with every response
  • Fallback hooks for human or agent review

Playbook for shipping ParseratorCore

Tabs below outline how teams prepare data, manage caches, and export plans without touching Firebase.

Prepare data before and after parsing

Chain the built-in preprocessors to normalize whitespace and schema keys, then let postprocessors prune empty outputs for downstream automations.

  • preprocessors.ts trims, cleans, and emits telemetry before Architect planning.
  • postprocessors.ts collapses null tokens and tidies JSON responses automatically.

Keep plans fresh with minimal effort

Use createInMemoryPlanCache and parseMany to skip duplicate Architect calls, while session.getAutoRefreshState() confirms when auto-refresh rebuilt a plan.

  • session.refreshPlan() or scheduled auto-refresh prevents stale instructions.
  • Cache contracts are shared across API, SDK, and local ParseratorCore instances.

Customize profiles and export everything

Toggle heuristics with applyProfile, capture results with createSessionFromResponse, and package migrations via session.exportInit().

  • Profiles set resolver mixes and fallbacks for lean-agent, vibe-coder, or sensor-grid modes.
  • Exported session init data keeps EMA promises when customers leave.

Operational guardrails keep teams confident

Parserator mirrors telemetry across the API, SDK, and dashboard so every parse can be traced—from the Architect’s SearchPlan to the Extractor’s deterministic passes and lean fallbacks.

Confidence metadata

Responses include metadata.confidence, token usage, and field:fallback events that match what Clear Seas dashboards display, making field-level audits straightforward.

  • Check provenance for every extracted field
  • Share telemetry downstream via custom listeners

Session snapshots

ParseratorSession records session.snapshot() history, automatically refreshes plans when confidence dips, and exposes getAutoRefreshState() so ops teams see when recalibration triggered.

  • Keep plan caches aligned across deployments
  • Export snapshots for compliance or incident reviews

Lean queue diagnostics

Async resolvers publish queue metrics, cooldown timers, and last-error metadata via getLeanOrchestrationSnapshot(), so fallbacks stay transparent and controllable.

  • Monitor fallback health in Clear Seas dashboards
  • Trigger alerts when queues approach limits

Integrate Parserator quickly

Use the REST endpoint or the Node SDK. Both expose the same safeguards and EMA commitments.

REST endpoint

curl -X POST "https://app-5108296280.us-central1.run.app/v1/parse" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer pk_live_your_api_key" \
  -d '{
    "inputData": "Acme Co. Contract Signed 3/12/24",
    "outputSchema": {
      "company": "string",
      "event": "string",
      "date": "date"
    }
  }'

Node SDK

npm install parserator-sdk

import { ParseratorClient } from 'parserator-sdk';

const client = new ParseratorClient({ apiKey: 'pk_live_your_api_key_here' });

const result = await client.parse({
  inputData: `John Smith\nSenior Developer\njohn@techcorp.com\n(555) 123-4567`,
  outputSchema: {
    name: 'string',
    title: 'string',
    email: 'email',
    phone: 'phone'
  }
});

console.log(result.parsedData);
