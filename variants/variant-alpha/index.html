How it stays explainable

Two-stage workflow with inspectable checkpoints

Parserator’s modular stages make it easy to trace every extraction decision, adjust heuristics, and re-run parses without rebuilding orchestration.

01

The Architect

Gemini 1.5 Flash inspects a small sample plus your schema, producing a reusable SearchPlan that flags ambiguities before extraction begins.

  • Reusable profiles
  • Schema aware
  • Telemetry-ready
02

The Extractor

Deterministic heuristics resolve most fields locally. Lean LLM calls only fire when the plan asks for assistance, reducing spend while maintaining accuracy.

  • Section-aware parsing
  • Deterministic fallbacks
  • Confidence scores
03

Telemetry & audit trail

Every request emits structured metadata so operators, agents, and customers can verify outcomes or rerun extractions with updated heuristics at any time.

  • Confidence reporting
  • Replay-ready
  • Freedom to leave

Integrate in minutes

Choose the surface that fits your automation

Whether you prefer raw HTTP calls or the Node SDK, Parserator keeps the experience consistent. Each example below mirrors the production surfaces running today.

Send JSON via REST

POST the data you want parsed alongside your desired schema. Parserator replies with structured JSON plus metadata describing every extraction decision.

  • Include schema + raw text in a single call
  • Confidence and token telemetry in the response
  • Backed by 95% accuracy and ~2.2s latency
curl -X POST "https://app-5108296280.us-central1.run.app/v1/parse" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer pk_live_your_api_key" \
  -d '{
    "inputData": "Acme Co. Contract Signed 3/12/24",
    "outputSchema": {
      "company": "string",
      "event": "string",
      "date": "date"
    }
  }'
