How the pipeline works

Architect → Extractor timeline

We plan once with Gemini 1.5 Flash, then execute deterministic extraction. You keep full visibility into every decision, fallback, and confidence score.

  1. 01

    Architect planning

    Gemini 1.5 Flash drafts a SearchPlan tuned to your schema and data profile.

  2. 02

    Deterministic extraction

    The Extractor runs regex, classifiers, and heuristics before escalating to the LLM.

  3. 03

    Confidence + telemetry

    Responses include per-field confidence, token usage, and trace IDs for review.

  4. 04

    Continuous improvement

    Feed telemetry back to refine plans or adjust the deterministic heuristics.

Integrate Parserator

Pick the surface that fits your automation

Use REST for direct calls or the Node SDK for structured automations—both ship with telemetry and safeguards.

Send JSON via REST

POST the data you want parsed alongside your desired schema. Parserator replies with structured JSON plus metadata describing every extraction decision.

  • Include schema + raw text in a single call
  • Confidence and token telemetry in the response
  • Backed by 95% accuracy and ~2.2s latency

curl -X POST "https://app-5108296280.us-central1.run.app/v1/parse" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer pk_live_your_api_key" \
  -d '{
    "inputData": "Acme Co. Contract Signed 3/12/24",
    "outputSchema": {
      "company": "string",
      "event": "string",
      "date": "date"
    }
  }'
